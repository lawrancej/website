Looking for a work study position? Are you interested in hacking on existing open source software to improve the classroom experience? If so, I have some project ideas for you.

Project ideas

  • Logisim is a circuit simulator useful for teaching computer architecture. Its author has moved on to other projects, but I'd like to improve it.
  • Software developers use code review systems such as Gerrit, ReviewBoard, GitLab, and Phabricator to ensure quality. Code review can be useful in the classroom, but for students to review each other's code, the review must be anonymous (double-blind): a student shouldn't know who's code he/she is reviewing, and a student shouldn't know who reviewed his/her code. Modify a code review system to support anonymous code review (or modify CrowdRanker to support Git repositories).
  • The Git installer for Windows ought to configure the git user name and email address to make it easier for students to get started with git in the classroom. The installer is written with InnoSetup. Hint: this is the file that needs to change.
  • Git ought to have a way to periodically fetch automatically from all remotes.
